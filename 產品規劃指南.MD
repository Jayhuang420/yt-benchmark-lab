# YT 對標頻道研究情報室 — 產品規劃指南

## 1. 產品願景

**一句話描述**：讓 YouTube 創作者快速了解競爭格局，找到成長突破口。

**目標用戶**：
- 中小型 YouTuber 與內容創作者
- 頻道經營者與策略規劃人員
- MCN 機構運營人員

**核心價值主張**：零門檻對標分析 + 可行動的優化建議

---

## 2. 功能規格

### 2.1 頻道設定模組
- YouTube Data API Key 輸入（附顯示/隱藏切換，持久化至 localStorage）
- 「我的頻道」URL 輸入（支援多種格式：`@handle`、`/channel/UCxxx`、純 Channel ID）
- 「對標頻道」多行輸入區（支援 1–10 個競爭頻道 URL）
- 頻道清單管理（新增/刪除）
- 所有設定自動保存至 localStorage

### 2.2 數據擷取引擎
透過 Client-side fetch 呼叫 YouTube Data API v3：

| API 端點 | 用途 | 配額消耗 |
|----------|------|---------|
| `channels.list` (snippet, statistics) | 頻道名稱、頭像、訂閱數、總觀看數、影片數 | 1 單位 |
| `search.list` (type: video, order: date) | 各頻道最新 10 支影片 | 100 單位 |
| `videos.list` (snippet, statistics, contentDetails) | 影片觀看數、按讚數、留言數、時長 | 1 單位 |

**URL 解析器**：自動從各種 YouTube URL 格式提取 Channel ID。

### 2.3 儀表板總覽
- **頻道概覽卡片**：每個頻道一張卡片（我的頻道金色高亮，對標頻道中性色調）
- 顯示：頭像、名稱、訂閱數、總觀看數、影片數、頻道年齡
- **並排比較表格**：各指標橫向對比，附百分比差異指標（綠色 ↑ / 紅色 ↓）
- **視覺化圖表**：
  - 長條圖：訂閱數對比、總觀看數對比
  - 雷達圖：多維度綜合比較

### 2.4 影片深度對比
- 各頻道最新影片卡片（縮圖、標題、觀看數、按讚數、發布日期、時長）
- 標記各頻道表現最佳影片
- 點擊可在新分頁開啟原始影片

### 2.5 AI 洞察建議引擎
內建啟發式分析（無需額外 AI API）：

| 分析類別 | 分析內容 |
|---------|---------|
| 內容策略 | 競爭頻道熱門主題關鍵字分析，找出內容缺口 |
| 上傳頻率 | 各頻道上傳頻率對比，建議最佳發片節奏 |
| 互動優化 | 按讚率/留言率對比，找出高互動內容特徵 |
| 成長軌跡 | 訂閱/觀看比率分析，評估成長效率 |
| 速贏策略 | 低門檻高回報的具體優化行動建議 |

## 3. 技術架構

### 3.1 架構概覽
- **前端**：單一 HTML 檔案（All-in-one），CSS/JS 完全內嵌
- **圖表庫**：Chart.js（CDN 引入）
- **後端**：無後端，全部 Client-side API 呼叫
- **資料持久化**：localStorage

### 3.2 資料流
```
用戶輸入 API Key + 頻道 URL
       ↓
URL 解析器提取 Channel ID
       ↓
Client-side fetch → YouTube Data API v3
       ↓
數據處理 + 快取至 localStorage
       ↓
儀表板渲染（卡片 + 表格 + 圖表）
       ↓
啟發式分析引擎 → 生成 AI 洞察建議
```

### 3.3 localStorage 資料結構
```json
{
  "yt_api_key": "AIzaSy...",
  "my_channel_url": "https://youtube.com/@mychannel",
  "competitor_urls": ["url1", "url2", ...],
  "cached_data": {
    "channels": {...},
    "videos": {...},
    "last_fetched": "2026-03-31T12:00:00Z"
  }
}
```

## 4. API 配額與限制

YouTube Data API v3 每日配額：**10,000 單位**

### 單次完整分析消耗估算
| 項目 | 計算 | 消耗 |
|------|------|------|
| channels.list | 11 個頻道 × 1 單位 | 11 |
| search.list | 11 個頻道 × 100 單位 | 1,100 |
| videos.list | ~110 支影片 × 1 單位 | 110 |
| **合計** | | **~1,221** |

### 配額管理策略
- 數據快取機制：同一頻道 1 小時內不重複抓取
- 提供「上次更新時間」顯示
- 配額不足時顯示友善提示
- 建議用戶每日分析不超過 8 次完整刷新

## 7. 使用指南

### 7.1 如何取得 YouTube Data API Key
1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 建立新專案或選擇現有專案
3. 啟用「YouTube Data API v3」
4. 建立 API 金鑰（API Key）
5. 建議設定 API 金鑰限制（僅限 YouTube Data API）

### 7.2 首次設定步驟
1. 開啟網頁應用
2. 在「設定」區輸入 YouTube Data API Key
3. 輸入自己的頻道 URL
4. 貼入 1–10 個想對標的競爭頻道 URL
5. 點擊「開始分析」

### 7.3 常見問題
| 問題 | 解決方案 |
|------|---------|
| API Key 無效 | 確認已啟用 YouTube Data API v3 |
| 配額超限 | 等待隔日重置，或申請配額提升 |
| 頻道找不到 | 確認 URL 格式正確，嘗試使用 Channel ID |
| 數據未更新 | 點擊「重新整理」清除快取重新抓取 |
